Overview

The Mail Bot design system features a sophisticated dark mode interface optimized for email campaign management. This guide documents all design patterns, components, and styles used throughout the application.

Key Principles

  • check_circle Dark First: Designed for reduced eye strain during extended use
  • check_circle High Contrast: Clear visual hierarchy with strategic use of color
  • check_circle Consistent Spacing: Predictable layouts using Tailwind's spacing scale
  • check_circle Professional Aesthetic: Clean, modern interface suitable for business use

Spacing & Layout

Border Radius

Default: 0.25rem (4px)

lg: 0.5rem (8px)

xl: 0.75rem (12px)

full: 9999px

Common Spacing Patterns

  • Component padding: p-4 (16px), p-6 (24px), p-10 (40px)
  • Card padding: p-6 (24px) for content cards
  • Section gaps: gap-2 (8px) for tight, gap-4 (16px) for normal, gap-6 (24px) for spacious
  • Page margins: px-10 (40px) on desktop, px-4 to px-6 on mobile
  • Vertical rhythm: mb-4, mb-6, mb-8 for consistent spacing

Best Practices

  • arrow_right Always use the dark: prefix for dark mode styles
  • arrow_right Include transition-colors on interactive elements for smooth hover effects
  • arrow_right Use backdrop-blur-sm on overlays and sticky headers
  • arrow_right Maintain consistent spacing with Tailwind's default scale (4, 6, 8, 10, etc.)
  • arrow_right Use border-white/10 or border-slate-200/10 for subtle borders
  • arrow_right Apply truncate to button text to handle overflow gracefully

Responsive Design

  • arrow_right Use responsive prefixes: sm:, md:, lg:, xl:
  • arrow_right Adjust padding on mobile: px-4 sm:px-6 md:px-10
  • arrow_right Use grid auto-fit for responsive cards: grid-cols-1 md:grid-cols-2 lg:grid-cols-3
  • arrow_right Hide non-essential elements on mobile with hidden md:flex
